# Opencode API Gateway - 使用指南

## 项目概述

基于 FastAPI 构建的 Opencode API 网关，提供 RESTful API 接口，支持版本查询、模型管理、工作目录切换、对话（SSE流式输出）、历史记录管理等功能。

**特点：**
- 绑定 `0.0.0.0`，支持局域网访问
- 自动会话管理和持久化
- 支持上下文理解和多轮对话
- SSE 流式输出，类似 ChatGPT 体验

## 核心 API 端点

### 1. 系统信息

```bash
# 查询版本
curl http://localhost:8000/api/version
```

### 2. 模型管理

```bash
# 查询可用模型
curl http://localhost:8000/api/models

# 切换模型
curl -X POST http://localhost:8000/api/sessions/{session_id}/model \
  -H "Content-Type: application/json" \
  -d '{"model": "kimi-for-coding/k2p6"}'
```

### 3. 工作目录

```bash
# 设置工作目录
curl -X POST http://localhost:8000/api/sessions/{session_id}/workdir \
  -H "Content-Type: application/json" \
  -d '{"path": "C:\\Users\\xxx\\Desktop"}'
```

### 4. 会话管理

```bash
# 创建会话
curl -X POST http://localhost:8000/api/sessions

# 列出所有会话
curl http://localhost:8000/api/sessions

# 获取会话详情
curl http://localhost:8000/api/sessions/{session_id}

# 删除会话
curl -X DELETE http://localhost:8000/api/sessions/{session_id}
```

### 5. 对话

#### 非流式对话

```bash
curl -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "你好",
    "session_id": "xxx"
  }'
```

#### SSE 流式对话

```bash
curl -X POST http://localhost:8000/api/chat/stream \
  -H "Content-Type: application/json" \
  -d '{
    "message": "请介绍Python",
    "session_id": "xxx"
  }'
```

### 6. 历史记录

```bash
# 查询历史
curl http://localhost:8000/api/sessions/{session_id}/history

# 清空历史
curl -X DELETE http://localhost:8000/api/sessions/{session_id}/history
```

## 完整使用流程

### 场景：在指定目录下工作并进行多轮对话

```bash
# 1. 创建会话并指定工作目录
SESSION=$(curl -s -X POST http://localhost:8000/api/sessions \
  -H "Content-Type: application/json" \
  -d '{"work_dir": "C:\\Users\\xxx\\Documents"}' | jq -r '.session_id')

echo "会话ID: $SESSION"

# 2. 切换模型
curl -X POST http://localhost:8000/api/sessions/$SESSION/model \
  -H "Content-Type: application/json" \
  -d '{"model": "kimi-for-coding/k2p6"}'

# 3. 进行对话
curl -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -d "{\"message\": \"列出当前目录文件\", \"session_id\": \"$SESSION\"}"

# 4. 继续对话（自动保留上下文）
curl -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -d "{\"message\": \"找出最大的文件\", \"session_id\": \"$SESSION\"}"

# 5. 查看历史记录
curl http://localhost:8000/api/sessions/$SESSION/history
```

## Python 客户端示例

```python
import requests
import json

BASE_URL = "http://localhost:8000"

# 创建会话
r = requests.post(f"{BASE_URL}/api/sessions", 
    json={"work_dir": "C:\\Users\\xxx\\Documents"})
session_id = r.json()["session_id"]

# 设置模型
requests.post(f"{BASE_URL}/api/sessions/{session_id}/model",
    json={"model": "kimi-for-coding/k2p6"})

# 对话
r = requests.post(f"{BASE_URL}/api/chat",
    json={"message": "你好", "session_id": session_id})
print(r.json()["response"])

# 流式对话
r = requests.post(f"{BASE_URL}/api/chat/stream",
    json={"message": "请介绍Python", "session_id": session_id},
    stream=True)

for line in r.iter_lines():
    if line:
        line = line.decode('utf-8')
        if line.startswith('data: '):
            data = json.loads(line[6:])
            if data.get('type') == 'text':
                print(data['content'], end='', flush=True)
```

## 配置说明

### 修改端口

编辑 `opencode_api.py`，修改最后一行：

```python
uvicorn.run(app, host="0.0.0.0", port=8080)  # 改为8080
```

### 修改会话存储位置

编辑 `SessionManager` 类的 `__init__` 方法：

```python
self.history_dir = "你的路径"
```

## 注意事项

1. **opencode 安装**：确保 opencode 已安装并在系统 PATH 中
2. **PowerShell**：Windows 系统下使用 PowerShell 执行命令
3. **防火墙**：如需局域网访问，请确保防火墙允许 8000 端口
4. **工作目录**：某些目录（如 Desktop）可能包含大量文件，会影响 opencode 响应速度
5. **编码**：API 返回 UTF-8 编码内容，请确保客户端正确解码

## 故障排查

### 服务无法启动

```bash
# 检查端口占用
netstat -ano | findstr :8000

# 检查 opencode 是否安装
opencode --version
```

### 局域网无法访问

1. 检查防火墙设置
2. 确认服务绑定到 0.0.0.0
3. 检查网络连接

### 对话无响应

1. 检查 opencode 是否正常工作
2. 尝试切换工作目录
3. 检查会话历史是否过长
